# ShipVitals interview

Ask these questions only when the project does not already have a clear product promise or `.shipvitals-config.json`.

1. What type of project is this? Web app, SaaS, API, CLI, Shopify app, extension, mobile app, automation, or client project?
2. Who is the target user or client?
3. What are the 3 to 5 critical flows that must work before release?
4. Where is this going next: public launch, client delivery, marketplace submission, internal use, or demo?
5. What stack and commands should be used to build, test, and run it?

Write the answers into `.shipvitals-config.json` before the next deep audit.
