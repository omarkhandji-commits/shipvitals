# Editorial standard

ShipVitals should read like a tool made by a careful maintainer, not like a launch thread.

## Use

- plain English;
- short explanations;
- direct claims;
- evidence before confidence;
- specific tradeoffs;
- concrete commands and file paths.

## Avoid

- inflated percentages;
- inflated superlatives and launch-thread language;
- repeating the same tagline in every file;
- claiming a check passed when it was not executed;
- vague advice such as "make it better" or "improve UX";
- long motivational language.

## Preferred style

Bad:

> Avoid absolute claims. Say what was checked, what passed, and what was not verified.

Good:

> ShipVitals is a release gate. It returns a verdict only after collecting evidence.
