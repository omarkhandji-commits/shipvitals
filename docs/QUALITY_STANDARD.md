# Quality Standard

ShipVitals qualifies for public release when each dimension below has current evidence.

| Dimension | Required evidence |
|---|---|
| Product value | Real projects expose release risks that normal command checks miss |
| Differentiation | Comparison documents state where ShipVitals fits beside specialist tools |
| Execution | Scripts, schemas, reports, CI, and package entry points pass validation |
| Repository | README, license, security policy, support path, changelog, and examples are current |
| Trust | Reports cap the score when runtime, visual, CI, or independent proof is absent |

## Release Rules

- Require evidence before `READY`.
- Name the file, command, or observed flow behind each finding.
- Require user confirmation before destructive actions.
- Mark untested areas `NOT VERIFIED`.
- Require runtime proof for client, marketplace, payment, auth, and data flows when applicable.
