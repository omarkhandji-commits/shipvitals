# Evaluation Method

A strong final gate must be evaluated on adversarial projects, not only clean demos.

## Evaluation set

- assistant-built SaaS with fake integration
- Landing page with broken mobile and placeholder copy
- API/CLI with weak validation and incomplete docs
- Shopify app with missing billing/privacy/review readiness
- Client dashboard with handoff and support gaps

## Success criteria

- detects blockers;
- refuses false READY;
- gives smallest fixes;
- produces retest commands;
- generates machine-readable evidence;
- keeps report understandable for non-senior builders.
