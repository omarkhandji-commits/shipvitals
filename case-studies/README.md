# Case Studies

These examples show the type of evidence ShipVitals expects. They are lightweight examples, not fake benchmark claims.

- `lab-ai-saas-dashboard`: catches demo data, missing auth proof, weak mobile state.
- `lab-shopify-app`: catches marketplace/privacy/scope gaps.
- `lab-api-cli`: demonstrates a project that can reach READY with deterministic CLI evidence.
- `real-open-source-project-1`: pypa/sampleproject real repository audit.
- `real-open-source-project-2`: tj/commander.js real repository audit.
- `real-open-source-project-3`: jonschlinkert/is-number real repository audit.
- `real-before-after`: real ShipVitals improvement log from local validation and real-world benchmark runs.
- `shipvitals-self-audit`: ShipVitals audited by its own release gate.
- `shipvitals-independent-review`: pending L6 independent review required before stable release.
